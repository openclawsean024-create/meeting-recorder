## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 企業中高階主管 | ~5 萬 | 每天 3-5 個會議、無法每個都做筆記 |
| 業務 / 客戶經理 | ~10 萬 | 客戶會議需紀錄但手寫跟不上 |
| 律師 / 會計師 | ~3 萬 | 需逐字稿但記錄耗時 |
| 個人 / 接案者 | ~30 萬 | 客戶會議需要可搜尋的紀錄 |

### 1.2 為什麼不做替代方案

- **手寫筆記**：跟不上會議速度、易遺漏重點
- **商用會議記錄工具（Otter.ai、Fireflies.ai）**：月費 10-30 USD，敏感會議資料外流風險
- **自己錄音後人工整理**：1 小時會議需 2-3 小時整理
- **我們的解法**：錄音 → STT 自動轉逐字稿 → AI 摘要重點 + 待辦，純前端可選雲端，零月費

---

## 2. 解決方案

### 2.1 核心價值主張

> 「1 小時會議 → 5 分鐘看完摘要重點 + 待辦 + 決策事項。」

### 2.2 使用者流程

1. 上傳錄音檔（mp3/wav/m4a）或即時錄音
2. STT 自動轉逐字稿（中文/英文/日文）
3. AI 自動分析：摘要重點 / 決策事項 / 待辦 / 與會者發言
4. 編輯修正（人機協作）
5. 一鍵匯出會議紀錄（Markdown / PDF / Email）

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| 會議機密性（敏感內容外流） | 🔴 高 | 明確聲明不上傳雲端（純本地模式） |
| STT 辨識錯誤 | 🟠 中 | 提供編輯介面修正 |
| Whisper API 成本 | 🟠 中 | 用戶用量配額 |
| 與會者未授權錄音（法律） | 🔴 高 | 明確聲明使用者需取得與會者同意 |
